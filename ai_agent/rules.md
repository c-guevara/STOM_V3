# 코드 구현 및 수정 규칙

## 1단계 - 초기 계획 수집 및 선택사항 알림
- 요청한 작업의 초기계획를 작성하고 보여준다.
- 작성한 초기계획 중, 선택사항이 있으면 사용자에게 결정을 물어본다.
- 선택사항은 마우스로 쉽게 클릭해서 결정할 수 있도록 반드시 선택메뉴 형태로 보여준다.
- 선택사항 개별 목록에는 추천정도와 설명을 덧붙인다.
- 초기계획을 보고 서로 상의해서 구체적인 계획으로 수정한다.
- 구체적인 계획에는 반드시 코드의 간단 예시를 포함한다.
- 구체적인 계획작성이 완료되면 코드구현 및 추가수정제안 메뉴를 보여준다.
- 코드구현을 선택하기 전에는 절대 코드구현, 추가, 수정 작업을 하지 않는다.

## 2단계 - 코드 구현 전, 계획 기록 및 새브렌치 생성
- 코드구현 명령이 내려졌을때, 코드 수정 및 추가 전에 다음 사항을 준수한다.
- ai_agent/plan_working 폴더에 한글파일명.txt 형태로 구체적인 계획을 자세히 기록하여 저장한다.
- git 새브렌치 working를 생성한다.

## 3단계 - 코드 구현
- 생성한 working 브렌치에서 코드구현 작업을 진행한다.
- 코드는 항상 간결하고 깔끔하게 작성하며 속도우선을 고려하여 구현한다.
- 구현시 클래스 또는 함수마다 간단한 설명을 주석으로 추가한다.
- 코드구현이 완료되면 작성한 계획을 다시 읽어보고 계획대로 구현되었는지 자세히 검토한다.
- 누락없이 계획대로 코드구현이 완료될때까지 계획 다시 읽어보기와 코드수정을 반복한다.

## 4단계 - 코드 테스트
- 계획대로 코드가 모두 구현완료되면 별도의 테스트용 파일을 만든다.
- 테스트용 파일에는 반드시 로그 기록 코드를 포함시키고 ai_agent/log 폴더에 기록한다.
- 스크린샷이 필요하면 찍어서 ai_agent/log에 같이 기록한다.
- 테스트 시 오류가 발생하면 로그와 스크린샷을 검토하여 다시 코드를 수정한다.
- 오류가 발생하지 않을때까지 오류검토와 코드수정을 반복한다.

## 5단계 - 계획파일정리, 메인 브렌치로 병합, 완료보고서 제출
- 계획 재검토 및 테스트(오류)검토를 통과하여 코드 구현이 완료되면 작성했던 계획파일을 ai_agent/plan_completed 폴더로 이동시킨다.
- working 브렌치를 메인 브렌치로 병합한다.
- 작업 과정을 포함한 완료보고서를 작성하여 보여주고 작업을 마감한다.

# 전략 생성 규칙
- ai_agent/strategy.txt 파일에서 변수설명과 예제를 자세히 읽어본다.
- 변수설명과 예제를 바탕으로 초기계획을 작성하여 제출한다.
- 초기계획 제출 이후의 과정은 코드 구현 및 수정 규칙을 준수한다.
- 데이터 형태는 1초스냅샷과 1분봉 두가지이며 동시 사용이 불가능하다.
- 매수전략 작성 시 반드시 매수 = True 상태임을 유념하고 전략을 작성한다.
- 매도전략 작성 시 반드시 매도 = False 상태임을 유념하고 전략을 작성한다.
- 매수전략과 매도전략을 분리하여 작성한다.
- 생성한 전략은 ai_agent/strategy 폴더에 한글전략명_생성일자및시간.txt파일 형태로 저장한다.
